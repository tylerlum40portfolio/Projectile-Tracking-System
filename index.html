LiDAR Projectile Tracking System

Project portfolio by Tyler Lum and Michael Milner documenting the design, implementation, and testing of an STM32-based projectile tracking system. The system uses LiDAR distance sensors, embedded C firmware, UART communication, and pan/tilt motor control to detect moving objects and command a tracking mechanism in real time.

Project Overview

This project focused on creating a projectile tracking system that can detect a moving target using LiDAR sensors and use that sensor data to command a pan/tilt mechanism. The goal was to build a complete mechatronic system that combined sensing, embedded processing, motor control, and real-time decision making.

The system is centered around an STM32 microcontroller. The STM32 receives distance data from the LiDAR sensors, checks whether the readings are valid, determines whether an object is within the tracking region, and then sends commands to the pan and tilt motors. The project required both hardware integration and software organization because the sensors, motors, and communication interfaces all needed to work together reliably.

A major focus of this project was modular embedded software. Instead of placing all logic directly in main.c, the code was separated into reusable drivers and control modules. The LiDAR driver handles measurement frames and error checking, while the motor driver handles direction and PWM control for the pan and tilt axes. This made the project easier to debug and better suited for documentation using Doxygen.

Overall, the project demonstrates how a microcontroller-based system can sense a moving object, process the measurement data, and produce actuator commands in real time. The final documentation explains the mechanical design, electrical wiring, embedded firmware, and testing process used to build the projectile tracking system.

Build Guide

This section outlines the major steps required to assemble and test the LiDAR projectile tracking prototype. The exact mechanical layout can be adjusted, but the same general integration process should be followed.

1. Assemble the Pan/Tilt Structure

Mount the pan and tilt axes so the sensor or tracking head can rotate smoothly. The mechanism should be stiff enough to avoid vibration but light enough for the motors to move quickly.

2. Mount the LiDAR Sensors

Install the LiDAR sensor or sensors on the tracking structure. The sensor should face the expected projectile path and should not be blocked by wiring, brackets, or the frame.

3. Install the STM32 Controller

Mount the STM32 board where USB and debugging connections remain accessible. Keep the controller protected from moving parts and make sure the ground connection is shared with the motor driver and sensors.

4. Wire the Motor Driver

Connect the pan and tilt motor driver inputs to the STM32 PWM and direction pins. Confirm that each motor responds correctly before running the full tracking code.

5. Connect LiDAR UART Signals

Connect LiDAR power, ground, transmit, and receive lines as required. Confirm that voltage levels are compatible with the STM32 before powering the sensor.

6. Upload Firmware

Build and flash the STM32 firmware. Begin with simple tests such as motor-only motion and LiDAR-only distance readings before enabling full tracking behavior.

7. Calibrate and Test

Test the sensor at known distances, verify that invalid measurements are rejected, and tune the tracking response until the system moves smoothly toward the detected target.

Mechanical & Electrical Design

Mechanical Design Overview

The mechanical design is based on a pan/tilt tracking head. The pan axis rotates the system left and right, while the tilt axis adjusts the vertical aim. This layout allows the tracker to follow a projectile or moving object in two directions while keeping the sensing hardware mounted on one compact assembly.

Sensor placement is especially important because the LiDAR must have a clear field of view. The sensor should be mounted rigidly and aligned with the center of the tracking direction. Any loose mounting can cause vibration, measurement inconsistency, or poor tracking behavior.

Electrical Design

The STM32 controller receives LiDAR data over UART and controls the motors through PWM and GPIO outputs. The motor driver handles the higher-current motor load, while the STM32 provides only the command signals. A common ground is required between the STM32, sensor, power supply, and motor driver.

Major Hardware

  • STM32 microcontroller: main embedded controller for sensing and control
  • LiDAR distance sensor: measures target distance and detects objects
  • Pan motor: rotates the tracker horizontally
  • Tilt motor: rotates the tracker vertically
  • Motor driver: converts STM32 PWM and direction signals into motor power
  • Custom PCB or wiring board: organizes sensor, power, and motor connections

Software Implementation

Architecture

The firmware was written using a modular embedded C architecture. The main program initializes the STM32 peripherals, starts PWM outputs, initializes the LiDAR driver, and then repeatedly runs the tracking logic. Separating the code into drivers made the project easier to test and easier to document with Doxygen.

LiDAR Driver

The LiDAR driver is responsible for initializing the sensor interface, reading measurement frames, checking for communication errors, validating distance and signal strength, and returning status codes. This allows the rest of the program to use clean functions instead of directly handling every UART detail inside the main loop.

Motor Driver

The motor driver abstracts the pan and tilt motor commands. Each motor command is clamped to a safe range, converted into a direction, and then applied using the correct PWM channel. Positive and negative values correspond to opposite directions, while zero stops the motor.

Tracking Logic

The tracking logic uses LiDAR distance readings to decide whether an object is detected. When the object is within the valid tracking range, the firmware updates the pan and tilt motor commands to follow the target. If the measurement is invalid or no object is detected, the system can stop the motors or hold its previous state depending on the selected behavior.

UART Debugging

UART debugging was used to print sensor status, distance readings, motor commands, and error states during testing. This made it much easier to determine whether problems were caused by the sensor, wiring, control logic, or motor output.

Code Architecture and File-by-File Implementation

The firmware is organized into separate files so each subsystem has a clear responsibility. This structure also improves the Doxygen output because each file can be documented as its own module.

Main Integration File

main.c

The main file initializes the STM32 system clock, GPIO pins, UART peripherals, timers, PWM channels, LiDAR device structure, and motor driver. It acts as the integration layer that connects all hardware and software modules.

LiDAR Driver Files

LiDar.h

This header defines the LiDAR data structure, valid distance limits, status enum, and function prototypes. It is the public interface used by the rest of the firmware.

LiDar.c

This source file implements LiDAR initialization, frame reading, checksum or validity checks, object detection, and status-to-string conversion.

Motor Driver Files

motor_driver.h

This header provides the function prototypes for initializing and commanding the pan and tilt motors.

motor_driver.c

This file starts the PWM channels and implements PanMotor_Set(), TiltMotor_Set(), and Motors_Stop(). It converts signed speed commands into PWM duty cycle and direction signals.

Possible Tracking Files

tracker.h / tracker.c

These files can be used to separate the projectile tracking algorithm from the hardware drivers. This would keep target detection, error calculation, filtering, and control behavior in one focused module.

How the Files Work Together

The LiDAR driver produces valid distance data. The tracking logic interprets that data and decides what motor motion is needed. The motor driver then applies those commands to the pan and tilt axes. The main file connects these modules together and runs the real-time loop.

Analysis and Results

Overall Performance

The final prototype demonstrated the core behavior required for a projectile tracking system: reading LiDAR distance data, identifying valid measurements, detecting objects within a selected range, and commanding motor motion through the STM32.

LiDAR Measurement Performance

The LiDAR driver improved reliability by checking whether measurements were inside the valid distance range and whether the signal strength was acceptable. This prevented the tracking logic from reacting to clearly bad data.

Motor Response

The pan and tilt motors responded to signed speed commands from the firmware. The motor driver made the control logic cleaner because the rest of the program could command motion without directly setting every GPIO and PWM register.

Challenges Encountered

  • Sensor noise: LiDAR measurements can vary during fast motion or poor alignment
  • Timing: sensor reads and motor updates must happen fast enough for tracking
  • Motor tuning: aggressive commands can cause overshoot or vibration
  • Wiring: UART, power, ground, and PWM connections must be checked carefully
  • Integration: getting sensing and motor response to work together was harder than testing each piece separately

Future Improvements

  • Add filtering for smoother distance measurements
  • Add angle feedback using encoders or potentiometers
  • Use a PID controller for pan and tilt positioning
  • Improve projectile prediction using velocity estimation
  • Add live plots for distance, motor command, and tracking error
