Detailed Description

Implementation of the laser GPIO driver.

The laser module is controlled using the CubeMX-generated LASER_SIGNAL GPIO output. This file keeps laser control separate from the main tracking logic so the system can safely enable, disable, toggle, and test the laser output.

Initializes the laser subsystem.

Starting with the laser disabled is a safety measure. The tracking loop should only enable the laser after the camera target is centered and the LiDAR range confirms that the object is within the accepted distance.

Initializes the laser driver and forces the laser off.

The initialization routine forces the laser into a known safe state by turning it off during startup. This prevents the laser from accidentally enabling when the STM32 first powers on or resets.
